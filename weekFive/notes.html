Testing and Debugging

  • Errors usually caused by 1 of 3 things
    • System error - There is a problem with the system or external devices
    • Programmer error - The program contains incorrect syntax or faulty logic
    • User error - The user has entered data incorrectly
  • Programmers should use strict mode so their code isn't sloppy. It is implemented with 'use strict'; You can put this at the beginning of hte .js file, or inside individual functions. Best method is the wrap all code inside a self-invoking function that invokes strict mode.
  • Alert can be a helpful tool for debugging, but is usually discouraged today as better method exist now.
  • console.log() is helpful with debugging; you can also implement console.trace() which will log an interactive stack trace.
  • The debugger command is very useful. It creates breakpoints that pause the execution of the code and allow you to see what is happening in the code.
  • Error objects can be created to give a message indicating what went wrong.
  • Developing with test-driven development is a useful way to create new code.
    • Write tests (they will fail initially)
    • Write code to pass the tests
    • Refactor the code
    • Test refactored code
    • Write more tests for new features
